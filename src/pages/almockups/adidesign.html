Design Objectives

1.ADI Landing Page

Objectives:

  • Streamline database discovery options at landing page
  • Improve search relevancy gradually
  • Port in browse databases by title A to Z feature
  • Redesign browse databases by topics to imporve brwosablity and mobile display responsiveness

2. Search/Browse Results page

Key Objectives:

  • Improve Usablity
  • Add database outage indicator at the point of use
  • Add new database indicator to promote new addition
    • Add trial database and trial indicator with automatic scheduler to show and hide after expiration date.
  • Continue to provide context driven - report a problem feature

3. About Database Detail Page

Key Objectives:

  • Ensure a Term of Use field is available to provide use instruction when needed. For example (for password protection ADI resources)
  • Provide browse related databases by similarity, by the same vendor, and by same topics
  • Provide permarlink bookmark option for repeating visit to a same database
  • Provide email permalink information for save for later (similar to skyline and summon's email feature)
